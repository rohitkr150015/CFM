Create a full, production-ready, responsive frontend for a College Course File Manager System using HTML, CSS, Bootstrap 5, and Vanilla JS, based on the following database schema:

Database: coursefiledb
Tables: Institute, Department, Teacher, User, Notification, Activity_Log, Program, Branch, Semester, Course, Course_File, Approval, Course_Teacher, Heading, Document, Comment, Template

The frontend must fully support Teacher, HOD, and Admin roles, role-based dashboards, and professional SaaS-grade UI.

1️⃣ Role-Based Features
Teacher

Dashboard cards: My Courses, Uploaded Files, Pending Files, Recent Uploads

Sidebar: Dashboard, My Courses, Upload Files, Manage My Files, Notifications, Profile Settings

File upload (drag & drop), versioning, progress bar, supported types: PDF, DOCX, PPT, ZIP, JPG, PNG

View own courses, documents, comments, approvals

HOD (Teacher promoted)

Additional Sidebar: Department Overview, Teacher Performance, Approve Course Files, Department Reports, Assign Courses, Manage Teachers

Dashboard cards: Total Teachers, Pending Approvals, Department Courses, Monthly Activity

Approve/Reject uploads, view all department courses, analytics charts, activity logs, bulk actions

Admin

Full system access: Dashboard, Users, Roles & Permissions, Colleges, Departments, Courses, Files, Reports, Settings

Manage all users, teachers, HODs, courses, programs, institutes

Backup & restore UI

Add/Edit/Delete College (Institute) Details: Name, Code, Address, Email, Phone, Website

Assign HOD to department, manage programs and branches

Audit logs for all actions
